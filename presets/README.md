# presets

starting points for `templates/project-identity-template.md`. pick the one closest to what you're building, copy it into your project as `guidelines.md`, then edit.

| preset | use for |
|---|---|
| [`editorial.md`](./editorial.md) | marketing sites, launches, essays, portfolios, doc home pages |
| [`saas.md`](./saas.md) | dashboards, admin tools, settings, multi-tab product surfaces |
| [`utility.md`](./utility.md) | single-purpose tools, calculators, weather, status pages, auth flows |

presets are opinionated on purpose. the point is to remove the blank-page problem — every preset has already made the decisions a cold identity file leaves to the agent. override anything that doesn't fit your product.

if your project straddles two (e.g. a SaaS with a strong marketing site), keep two identity files — one per surface — and reference them from the routing layer.
