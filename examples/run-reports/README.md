# run report examples

illustrative examples for the docs-first explainability layer.

these are not final product contracts. they exist to test whether the report shape is:
- legible
- useful
- compact
- honest about human judgment

## examples

- [utilitarian app](./utilitarian-app-example.md)
- [dense dashboard](./dense-dashboard-example.md)
- [marketing / editorial](./marketing-editorial-example.md)

## what to look for

- can you tell why the preset matched?
- can you tell why some skills fired and others did not?
- can you see the difference between issues detected and fixes applied?
- is the human judgment boundary clear?
- does the report help you trust the system more?
